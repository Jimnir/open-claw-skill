# 面试冲刺·第1期（2026-06-27）

**目标岗位：** AI 工具链工程师（OpenClaw + Claude Code 方向）
**已完成：** 10道核心题练习 ✔️

## 第 1 题：OpenClaw 核心架构

**核心组件：**
- **Gateway（网关）** — 消息路由、Agent 调度、会话管理
- **Agent（智能体）** — 驱动大模型，通过 SOUL.md 定义人格
- **Plugins / Skills（插件与技能）** — 工具箱，Markdown 指令手册

**关键概念：** 消息进 → Agent 处理 → 工具调用 → 结果出

**回答公式：** 核心概念 + 展开说明 + 用过的东西 + 对比差异

---

## 第 2 题：Skills 工作原理与封装

**本质：** 用 Markdown 写的 Agent 操作手册。

**封装步骤：**
1. 建目录：`workspace/skills/分类/技能名/`
2. 写 SKILL.md：YAML 头（name/description）+ Markdown 正文（触发条件+步骤）
3. 验证：`openclaw skills list`
4. 测试：新会话触发

**加载优先级：** 工作区 > 本地（`~/.openclaw/skills`）> 捆绑

**进阶概念：** Skill Workshop（`skill_workshop` 自动化创建）、ClawdHub（社区技能注册表）、条件门控（`metadata.openclaw.requires`）

---

## 第 3 题：OpenClaw vs Claude Code 场景对比

| 维度 | OpenClaw | Claude Code |
|------|----------|-------------|
| 定位 | AI 助手网关/多 Agent 平台 | 终端 AI 编程助手 |
| 核心 | 消息路由→Agent→工具调用 | 看懂代码→写代码→执行命令 |
| 场景 | 自动化工作流、定时任务、消息处理 | 代码生成、重构、Debug |
| 交互 | Gateway 统一入口 | 终端 CLI 交互 |
| 扩展 | Skills Markdown 机制 | 文件读写+命令行执行 |

**实际搭配：** Claude Code 写代码干活 → OpenClaw 做调度和自动化。两者可通过 CCR 打通。

## 第 4 题：AI 日报自动化工作流

**亲手做的项目：** AI 行业日报定时推送

**设计流程：**
1. Cron 定时触发（每天 22:00，`"0 22 * * *"`）
2. `web_search` 自动搜中英文 AI 新闻（双源覆盖）
3. Agent 整理成结构化日报（今日头条 / 行业动态 / 技术前沿 / 工具更新）
4. 多路径输出：`memory/` + `D:\ai-daily\` + 聊天窗口推送

**关键：** 需求分析 → 定时触发 → 数据采集 → 信息处理 → 多路径输出

---

## 第 5 题：企业级技能库搭建

**四层规范：**
1. **仓库 README** — 定位说明 + 安装指引 + 环境要求
2. **分类组织** — `devops/`、`interview-prep/`、`tutoring/` 等子目录
3. **命名规范** — 小写连字符、description ≤160 字符、自解释
4. **每个技能双文档** — SKILL.md（给 Agent）+ README.md（给人）

**加分：** 仓库级使用规范文档，约束团队一致性；加载优先级实现公私分离。

---

## 第 6 题：TaskFlow 持久化任务

**定义：** OpenClaw 的持久化多步骤任务引擎

**对比普通会话：**

| 维度 | 普通会话 | TaskFlow |
|------|---------|----------|
| 寿命 | 一次回答 | 数小时/天 |
| 状态 | 用完就丢 | `stateJson` 持久化 |
| 等待 | 必须一轮完成 | `setWaiting` 等人/外部系统 |
| 多步骤 | 全靠上下文 | `currentStep` 管理 |
| 重启 | 结束就没了 | 可 `resume` 恢复 |
| 子任务 | 无 | `runTask` 管理子任务 |

**典型场景：** 审批流程（提交→等待审核→恢复执行→自动部署）

## 第 7 题：多 Agent 配置

**亲手做的：** 配置了 4 个 Agent
- 姬子（日常/教学）
- 代码助手（编程/调 Claude Code）
- 面试官（模拟面试）
- 百科全书（查资料）

**核心价值：** 职责分离 + 技能聚焦 + 灵活扩展 + 权限控制

**每个 Agent 独立：** 工作区 → 独立的 SOUL.md（人格） + 独立的 skills/（技能集）

## 第 8 题：docker-deploy 技能设计

**技能三功能：** ① 自动生成 Dockerfile ② 自动生成 docker-compose.yml ③ 部署配置检查

**核心价值：** 把 Docker 部署门槛降低，流程标准化，减少手动出错

**面试金句：** "我不会 Docker，但我会把 Docker 部署的规范写成 Skill。团队里别人也不会？没关系，Agent 会。"

---

## 第 9 题：OpenClaw 记忆体系

**三层架构：**
1. **MEMORY.md（长期记忆）** — 精华，跨会话持久
2. **memory/日报（短期记忆）** — 每日原始记录
3. **会话上下文（工作记忆）** — 用完即丢

**设计哲学：** "Write it down. Mental notes don't survive session restarts. Files do."

**维护策略：** 定期翻 daily notes → 提炼进 MEMORY.md → 删过时信息

## 第 10 题：车牌识别项目实战（STAR 法则）

- **S（背景）**：大作业，4人小组，OpenCV 模板匹配
- **T（任务）**：蓝牌+绿牌识别，含汉字
- **A（行动）**：
  - 中文显示：OpenCV 不支持 → 改用 Pillow + 中文字体 fallback
  - 汉字识别不准 → 字体生成标准模板 + 缓存机制
  - 绿牌识别 → 绿色通道提取优化
- **R（结果）**：蓝牌绿牌都支持，中文正确显示，加 CLI 参数

**追问应对：** 准确率可验证方案可行性；分工自己是核心技术主力。

**💡 参考框架：**

> 1. 部署了一套多 Agent 平台（OpenClaw），Claude Code 配了国内直连的路线

**练习方法：** 三段式结构（做了什么→具体例子→结果），先念顺再脱稿。

## 题目 2：OpenClaw 与传统 Bot Framework 的区别

**表现：** ✅ 抓住关键点（自动化 + 易用性），但深度不够

**💡 完整回答：**

> 最大的区别在于 **定位和架构**：
> 
> 1. **传统 Bot（Rasa / 微信对话平台）** 是"问答机器"——你给我一句话，我给你一个回复。本质是一问一答的对话系统。
> 
> 2. **OpenClaw 的定位是"AI 网关 + Agent 调度平台"**——它不止能聊天，还能：
>    - 跑定时任务（Cron 自动搜新闻推日报，没人管也能跑）
>    - 多步骤持久化任务（TaskFlow，审批流程可以等人回复再继续）
>    - 多 Agent 分工（日常助手 / 代码助手 / 面试官各干各的）
>    - Skill 封装（谁都能写 Markdown 给 Agent 加新能力）
> 
> 传统 Bot 是一个"嘴巴"，OpenClaw 是一个 **能调度、能干活、能自动跑的"操作台"**。

---

## 题目 3：Skill 机制解决了什么问题

**表现：** ❌ 需要大量引导才能说，核心理解到位但表达不出来

**💡 一句话版：**
> Skill 解决的本质上是一个问题：**把经验固化下来。** 你教 Agent 做一件事，它下次就会了。没有 Skill 的 Agent 像个失忆症患者，每次都要从头教。

**💡 完整版：**
> Skill 解决的最大问题是 **知识的可复用性**。
>
> 1. **不用每次都教** — 封装一次 docker-deploy，以后哪个项目要部署，Agent 自己就会
> 2. **经验标准化** — 团队里谁写 Skill，大家都能用，不会因为人走了知识就丢了
> 3. **降低门槛** — 写个 Markdown 就行，不用写代码，业务人员也能参与
>
> 没有 Skill？那每次都得给 Agent 写 prompt、调参数，跟重新训练一个实习生一样累。

---

## 题目 4：场景设计题 — 自动化周报系统

**表现：** ✅ 框架正确（定时 + 需求 + 输出），展开不够

**💡 完整回答：**
> 分 4 层设计：
> 1. **触发层** — Cron 定时，每周五 18:00 自动触发
> 2. **数据采集** — Agent 调 Claude Code 去跑 `git log --since="7 days ago"`，拉本周所有 commit
> 3. **整理输出** — Agent 把 commit 按类型分类（功能/修复/文档），写成 Markdown 周报
> 4. **推送存档** — 发企微群 + 存档到 `周报/YYYY-MM-DD.md`
>
> 成熟后封装成 weekly-report Skill，其他项目开箱即用。
