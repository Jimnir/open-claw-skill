# Python 期末助教 🐍

> Python 作业辅导、代码调试、知识点讲解、作业审查一站搞定。

## 技能简介

**python-tutor** 是你的私人 Python 助教，专为期末项目和作业设计。遇到 bug 不会调？知识点理解不透？作业写完了不确定对不对？找它就对了。

## 使用场景

| 场景 | 说明 |
|------|------|
| 🐛 **代码调试** | 贴出报错信息，帮你分析错误原因并给出修复方案 |
| ✏️ **代写代码** | 描述需求，生成完整可运行的代码（带中文注释） |
| 📖 **讲解知识点** | 用大白话加例子解释 Python 概念 |
| ✅ **检查作业** | 审查你的代码逻辑和风格，提出优化建议 |

## 触发方式

直接在对话中说出你的需求：

- **调试**：「报错了」「出 bug 了」「跑不通」+ 贴代码和报错信息
- **写代码**：「帮我写一个...」「我要实现...」
- **讲解**：「讲一下装饰器」「闭包是什么意思」
- **检查**：「帮我检查一下代码」「帮我看看作业」

## 工作模式

### 模式 1：Debug（调试）

```
你：我这段代码报错了…（附代码和错误信息）
→ 1. 分析错误类型（语法错误/运行时错误/逻辑错误）
→ 2. 指出具体错误位置和原因
→ 3. 给出修改后的代码
→ 4. 让你验证修改结果
```

### 模式 2：Write（写代码）

```
你：帮我写一个读取 CSV 文件并统计每列平均值的程序
→ 1. 确认输入输出格式
→ 2. 生成完整可运行的代码（中文注释）
→ 3. 解释每段代码的作用
→ 4. 提醒注意事项
```

### 模式 3：Explain（讲知识点）

```
你：讲一下 Python 的装饰器
→ 1. 用生活中的例子解释概念
→ 2. 一段简单的代码演示用法
→ 3. 检查你是否理解了
→ 4. 没懂就换个角度再讲一遍
```

### 模式 4：Review（检查作业）

```
你：帮我检查一下这个作业代码
→ 1. 检查代码逻辑是否正确
→ 2. 指出潜在问题
→ 3. 如果有更好的写法，给出改进方案并解释原因
```

## 使用建议

1. **贴完整的错误信息**：调试时把完整的 traceback 贴出来，不要只贴最后一行
2. **先自己尝试**：遇到问题先自己 debug 10 分钟，实在搞不定再找助教
3. **理解 > 复制**：别光复制代码，要搞懂每一行在干什么
4. **追问无妨**：没听懂就直说，助教不会嫌你烦
